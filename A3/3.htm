Assignment 3 Problem 5. [20 Marks correctness. File: roots.c]

The Bisection method is used to find a root of a given function f(x) within a given interval (p,q) (exclusive range). For this problem we will assume that f(x) is a quadratic function of the form f(x)=a*x*x+b*x+c. The general concept for Bisection method is similar to the binary search and can be explained as follows:

Initially the interval (p,q) contains a root, i.e., f(x) intersects the x-axis. Hence, sign(f(p)) != sign(f(q)) (see the Figure.1 for explanation). We now find the mid-point, say r, in the interval, i.e., r=(p+q)/2. Then we search within a smaller interval based on the sign of f(r). If sign(f(r)) == sign(f(p)), we search within (r, q). If sign(f(r)) == sign(f(q)), , we search within the interval (p, r). We reach a solution when f(r) == 0.

For this problem, we reach a solution when fequals(f(r),0) (see problem 0b above).

Write a C module (roots.c) that provides the function:
float find_root(float a, float b, float c, float p, float q);
find_root takes the coefficients a,b,c of a quadratic equation and an interval (p, q). It will return the root of this equation in the given interval. You may safely assume that there will be exactly one root in the given interval. Also you may assume that initially f(p) != 0 and f(q) != 0.

For example:
find_root(1, -8, 15, 2, 4) should produce a root "close to" 3.0
find_root(4, -56, 75, 0.5, 3.0) should produce a root "close to" 1.5

Assignment 3 BONUS Question. [up to 5 Marks]

A Fibonacci word is composed by repeatedly concatenating words composed of binary digits. The first two words are defined to be 0 and 01, respectively. Each next word is the concatenation of two previous ones. So, the third word is 010, the fourth 01001 etc: "0, 01, 010, 01001, 01001010, ..."

In this question we will consider sequences created the same way but using different starting words. For example, the first two words 10 and 100 produce "10, 100, 10010, 10010100, 1001010010010...". We will use integers to represent binary words, and will only consider words that begin with 1. For example, 2 represents the word 10 and 6 represents the word 110.

Your task is to write a function int fib_word(int s0, int s1, int word) that checks whether word can be generated using the above rules from s0 and s1. Write a C module (fib_word.c) that provides the following function:

// fib_word(s0,s1,word) returns true if word is part of a sequence where
//     s0 and s1 represent the first two elements, and the
//     n-th element is formed by concatenating binary representations
//     of elements at (n-1) and (n-2). If word is not a part of
//     that sequence, false is returned.
// requires: s0 > 0, s1 > 0, word > 0
bool fib_word(int s0, int s1, int word);

Public test samples:
