Assignment 2 Problem 3. [25 Marks for correctness]

Write the C implementation date.c for the interface date.h.

Public test samples:

const struct date date1 = {33,12,2013};
const struct date date2 = {23,12,2013};
const struct date date3 = {10,2,2010};
const struct date date4 = {23,12,2013};
const struct date date5 = {29,2,2100};
const struct date date6 = {10,12,2013};

assert(is_valid(date1)==false);
assert(d_eq(date4 ,date2)== true);
assert(is_valid(date5)==false);
assert(greaterthan(date2,date3)== true);
assert(day_number(date3)==41);
assert(days_between(date6, date4)==13);

Notes:

You may assume that the dates will be after January 1, 1900; i.e. we will not test your code using dates before this and you do not need to assert this restriction.

3 criteria must be taken into account to identify leap years (years where February has 29 days):

  1. If the year is evenly divisible by 4 then it is a leap year.
  2. If the year is evenly divisible by 100 then it is NOT a leap year (this takes precedence over rule 1).
  3. If the year is evenly divisible by 400, then it is a leap year (this takes precedence over rule 2).
